# Agent Brief

## Startup
- Read only `docs/agent_state.md`, `docs/agent_brief.md`, and the last 30 lines of `docs/work_log.md`.
- Continue from `Next Action`; do not repeat completed work.
- Open `project_memory.md`, `gdd.md`, `tdd.md`, or `plan.md` only when the current task requires their specific content.

## Workflow
- Keep changes small, testable, and aligned with the existing architecture.
- Do not redesign unrelated systems or overwrite unrelated user changes.
- Record stable decisions in `project_memory.md`, progress in `agent_state.md`, and a short chronological note in `work_log.md`.
- Keep memory, state, log, and full design documents separate from this brief.

## Completion
- Update `agent_state.md` with completed work, blockers, and the next recommended action.
- Report what changed, files modified, tests run, and what remains.
