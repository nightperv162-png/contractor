# Agent Rules Short

Always:
- Read /docs/agent_state.md first.
- Read only the last 30 lines of /docs/work_log.md unless needed.
- Use existing architecture.
- Keep changes small and testable.
- Use centralized config.
- No magic numbers outside config.
- Canvas-only player-facing UI.
- Do not redesign unrelated systems.
- Update /docs/agent_state.md after work.
- Append a short entry to /docs/work_log.md.
- Run tests/build if code changed.
- Commit only after checks pass.
- Final report should be short.
